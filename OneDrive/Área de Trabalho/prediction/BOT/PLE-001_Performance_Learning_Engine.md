# Purpose

The Performance Learning Engine transforms historical performance into actionable knowledge. It identifies why performance changes, validates improvements through replay and paper trading, and proposes optimizations without automatically modifying live trading.

# Responsibilities

- Analyze historical performance
- Detect improving and degrading behaviors
- Generate optimization candidates
- Compare champion vs challenger strategies
- Recommend retraining
- Feed the Trading Knowledge Brain
- Supply datasets to the AI Research Lab

# Core Philosophy

The Trading OS should always learn but should never change live behavior without validation and user approval.

# Learning Pipeline

1. Collect performance history
2. Detect patterns
3. Identify root causes
4. Generate hypotheses
5. Validate via replay
6. Validate via paper trading
7. Publish recommendation
8. Await user approval for live promotion
