# Purpose

This manual defines how end users operate the Trading OS safely across Development, Replay, Paper Trading, Evaluation, Funded, and Live modes.

# Operating Principles

- Risk protection is always active.
- Replay before Paper.
- Paper before Live.
- Live trading requires explicit user confirmation.
- All actions are audited.

# User Modes

- Development
- Replay
- Paper Trading
- Evaluation
- Funded
- Personal Live

# Daily Workflow

1. Verify system health
2. Verify broker connectivity
3. Review market intelligence
4. Review news and macro events
5. Verify account status
6. Enable desired trading mode
7. Monitor execution
8. Review journal
9. Review performance
10. End-of-day archive

# Emergency Procedures

- Activate Kill Switch
- Pause Live Trading
- Continue Paper Trading if desired
- Review incident log
- Resume only after validation

# Best Practices

- Keep Replay datasets updated.
- Review journal after every session.
- Validate new strategies in Replay and Paper.
- Never disable risk protections.
- Keep broker connections synchronized.
